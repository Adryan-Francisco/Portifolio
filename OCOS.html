Componentes Fundamentais de um Computador

CPU (Unidade Central de Processamento)

  • Executa instruções e operações
  • Contém ULA, registradores e unidade de controle
  • Ciclo básico: Busca, Decodificação, Execução, Armazenamento

Memória

  • Hierarquia: Cache, RAM, Armazenamento secundário
  • RAM (volátil) e ROM (não-volátil)
  • Memória virtual para expandir capacidade

Dispositivos de E/S e Barramentos

  • Periféricos de entrada e saída
  • Barramentos de dados, endereços e controle
  • Interfaces como USB, HDMI, SATA

Hierarquia de Memória

A hierarquia de memória organiza os diferentes tipos de memória em termos de velocidade, capacidade e custo, otimizando o desempenho do sistema.

Tipos de Memória

  • Registradores: Mais rápidos, dentro da CPU
  • Cache (L1, L2, L3): Memória rápida para dados frequentes
  • RAM: Memória principal volátil
  • ROM: Memória não-volátil para firmware
  • Armazenamento secundário: Discos rígidos, SSDs

Características

Tipo Velocidade Capacidade Custo
Registradores 1-2 ns KB Alto
Cache 2-10 ns MB Alto
RAM 10-100 ns GB Médio
Armazenamento ms TB Baixo

Memória Virtual

Técnica que usa espaço em disco como extensão da RAM, gerenciada pelo sistema operacional através de paginação.

Unidade Central de Processamento (CPU)

Componentes da CPU

  • ULA (Unidade Lógica e Aritmética): Executa operações matemáticas e lógicas
  • Unidade de Controle: Coordena as operações
  • Registradores: Armazenamento temporário rápido
  • Cache: Memória rápida para dados frequentes

Ciclo de Instrução

  1. Busca (Fetch): Obtém instrução da memória
  2. Decodificação (Decode): Interpreta a instrução
  3. Execução (Execute): ULA realiza operação
  4. Armazenamento (Store): Salva resultados

Unidade Lógica e Aritmética (ULA)

Operações Aritméticas

  • Adição, subtração
  • Multiplicação, divisão
  • Operações com complemento para dois

Operações Lógicas

  • AND, OR, NOT, XOR
  • Comparações (maior, menor, igual)
  • Deslocamento e rotação de bits

Entradas e Saídas

  • Operandos (dados de entrada)
  • Código de operação (da unidade de controle)
  • Resultados e códigos de condição (overflow, zero, etc.)

Técnica de Pipeline

O pipeline permite a execução sobreposta de instruções, dividindo o processamento em estágios que podem operar simultaneamente em diferentes instruções.

Estágios do Pipeline

1. Busca (Fetch)

Carrega a próxima instrução da memória para o registrador de instrução

2. Decodificação (Decode)

Interpreta a instrução e prepara operandos

3. Execução (Execute)

ULA realiza a operação especificada

4. Acesso à Memória

Lê ou escreve dados na memória se necessário

5. Escrita (Write Back)

Armazena resultados em registradores ou memória

Desafios do Pipeline

Hazards (Perigos)

  • Data Hazard: Dependência de dados entre instruções
  • Control Hazard: Mudanças no fluxo de controle (desvios)
  • Structural Hazard: Conflito por recursos

Soluções

  • Forwarding (adiantamento de resultados)
  • Branch Prediction (predição de desvios)
  • Execução fora de ordem
  • Pipeline superescalar (múltiplas pipelines)

Barramentos e Comunicação

Barramentos são vias de comunicação que permitem a transferência de dados entre os componentes do computador.

Tipos de Barramentos

Barramento de Dados

  • Transfere dados entre componentes
  • Largura determina quantidade de bits transferidos simultaneamente
  • Ex: 32 bits, 64 bits

Barramento de Endereços

  • Especifica localizações de memória
  • Largura determina memória máxima endereçável
  • Ex: 32 bits → 4GB, 64 bits → 16 exabytes

Barramento de Controle

  • Transporta sinais de controle
  • Coordena operações (leitura, escrita, interrupções)
  • Sincroniza comunicações

Exemplos de Barramentos

Barramento Tipo Uso Características
PCI Expansão Placas internas Paralelo, substituído pelo PCIe
PCI Express Expansão Placas de vídeo, etc. Serial, alta velocidade, múltiplas lanes
USB E/S Periféricos externos Hot plug, versões 2.0, 3.0, etc.
SATA Armazenamento HDDs, SSDs Serial, substituiu IDE

Dispositivos de Entrada e Saída

Tipos de Dispositivos

  • Entrada: Teclado, mouse, scanner, câmera
  • Saída: Monitor, impressora, alto-falantes
  • Armazenamento: HD externo, pendrive
  • Comunicação: Modem, roteador

Processo de Comunicação

  1. Conexão física (cabo ou sem fio)
  2. Detecção e instalação de drivers
  3. Transferência de dados gerenciada por módulos de E/S
  4. Monitoramento de status e desempenho

Desafios

Compatibilidade

Garantir funcionamento com diferentes sistemas e versões

Desempenho

Evitar gargalos e conflitos

Segurança

Proteger contra ameaças por dispositivos externos
