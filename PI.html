Visão Geral

O Projeto Integrador I visa aplicar conhecimentos adquiridos nas disciplinas iniciais do curso de ADS, através do desenvolvimento de soluções tecnológicas para problemas reais ou simulados.

Neste projeto, utilizaremos o Git e GitHub para controle de versão e colaboração em equipe, essencial para qualquer desenvolvedor profissional.

Controle de Versionamento com Git

Git é um sistema de controle de versão distribuído criado em 2005 por Linus Torvalds. Principais vantagens:

  • Gratuito e open source
  • Alta performance e escalabilidade
  • Operações locais (sem necessidade de servidor sempre disponível)
  • Integridade garantida dos dados com checksum SHA-1
  • Suporte a diferentes workflows e colaboração eficiente

Git vs GitHub

Git é o sistema de controle de versão distribuído, enquanto GitHub é um serviço de hospedagem de código que utiliza Git. Eles são ferramentas diferentes e complementares!

Configuração Inicial do Git

Para começar a usar o Git, é necessário configurar seu usuário:

$ git config --global user.name "Seu Nome" $ git config --global user.email "seu@email.com"

Para verificar as configurações:

$ git config --list

Fluxo Básico de Trabalho

O Git trabalha com três áreas principais:

  1. Working Directory: Arquivos modificados, inseridos ou excluídos
  2. Staging Area: Arquivos rastreados e prontos para serem versionados
  3. Repository: Alterações efetivadas e registradas

Comandos Essenciais:

$ git init # Inicializa um repositório local $ git status # Verifica o estado dos arquivos $ git add . # Adiciona todas as alterações $ git commit -m "msg" # Registra as alterações com mensagem $ git log # Mostra o histórico de commits

Trabalhando com Branches

Branches são ramificações no projeto usadas para desenvolver funcionalidades sem impactar a versão principal (main).

Comandos para Branches:

$ git branch # Lista branches $ git branch nome_branch # Cria nova branch $ git checkout nome_branch # Muda para a branch $ git merge nome_branch # Mescla branches

Boa Prática

Mantenha a branch principal (main/master) estável e sempre funcional. Crie branches separadas para cada nova funcionalidade ou correção de bugs.

Trabalhando com GitHub

GitHub é uma plataforma web para hospedar repositórios Git e colaborar em projetos com a comunidade.

Fluxo Básico:

  1. Crie um novo repositório no GitHub
  2. Conecte seu repositório local ao remoto: $ git remote add origin URL_DO_REPOSITORIO
  3. Envie suas alterações para o servidor: $ git push origin main
  4. Para baixar atualizações do repositório remoto: $ git pull origin main

Trabalho em Equipe com Pull Requests

Pull Requests (PRs) permitem revisão de código e discussão antes de mesclar alterações na branch principal:

  1. Crie uma branch para sua funcionalidade
  2. Faça os commits necessários
  3. Envie para o GitHub (git push)
  4. Abra um Pull Request no GitHub
  5. Após revisão e aprovação, faça o merge

Resolvendo Conflitos

Quando há alterações conflitantes em commits diferentes, é necessário editar manualmente os arquivos marcados pelo Git antes de completar o merge.

Boas Práticas em Git

  • Commits atômicos: Uma mudança lógica por commit
  • Mensagens claras: Descreva o que foi feito de forma concisa
  • Use .gitignore: Não versione arquivos desnecessários (node_modules, .env, etc)
  • Sincronize frequentemente: Evite conflitos fazendo pull regularmente
  • Revise o código: Avalie PRs antes de fazer merge
  • Documente mudanças: Mantenha um README atualizado

Etapas do Projeto

  1. Escolha do Tema / Problema: Defina o escopo do projeto
  2. Análise e Planejamento: Setup do repositório e ferramentas
  3. Modelagem: Versionamento de documentos e diagramas
  4. Prototipagem: Usando branches para cada funcionalidade
  5. Documentação: README completo e comentários no código

Conclusão

Este projeto representa o início da jornada prática dos estudantes no mundo da tecnologia. O domínio de Git e GitHub é essencial para:

  • Trabalhar eficientemente em equipes
  • Organizar e rastrear o código
  • Colaborar com desenvolvedores ao redor do mundo
  • Manter histórico completo das mudanças

Estas habilidades são requisitos fundamentais para qualquer desenvolvedor de software profissional no mercado atual.
